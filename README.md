# Lightning2EarthCARE

**Lightning2EarthCARE** is a workflow for collocating **EarthCARE satellite observations** with **satellite lightning measurements** and building a catalogue of lightning storms observed during EarthCARE overpasses.

The repository contains four main components:

## lightning2ec
The `lightning2ec` component implements the **core collocation pipeline**.

It:
- queries **EarthCARE MSI and CPR data** from the ESA MAAP STAC catalogue
- (down)loads **MTG-LI and GOES-GLM lightning observations**
- performs spatial–temporal matching between EarthCARE and lightning detections
- clusters lightning groups and produces collocated datasets

Detailed documentation and usage instructions are available in:
`lightning2ec/README.md`

## Storm catalogue generation
The script:
`create_storm_catalogue.py`
processes the outputs generated by `lightning2ec` and builds a **catalogue of lightning storms** that were overpassed by the EarthCARE satellite.

The catalogue summarizes lightning storm properties and provides a dataset for further analysis with EarthCARE data.

## Postprocessing tools
The `postprocessing/` directory contains scripts for working with pipeline outputs and logs.

These scripts are used to:
- parse pipeline **log files**
- generate **flags for missing or incomplete data**
- convert outputs to **Parquet format** for more efficient storage and analysis

## Example notebooks
The `notebooks/` directory contains Jupyter notebooks demonstrating how to work with the pipeline outputs:
- **opening the data** – loading collocated datasets and the storm catalogue
- **plotting** – visualising lightning data, together with EarthCARE overpasses
